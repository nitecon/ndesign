Testing Push Connections

The WebSocket feed and SSE streams above connect to the Go test server at http://localhost:28080. Ensure the server is running before loading this page.

WebSocket — connects to ws://localhost:28080/ws/feed. The server should push JSON messages with fields: time, symbol, price, volume, type.

SSE — connects to /events/stream. The server should emit named events:

  • deployment — fields: version, environment, status
  • metric — fields: cpu, memory, disk

REST Endpoints:

  • GET /api/stats — returns version, uptime, cpu_usage
  • GET /api/stats/live — returns cpu_usage, active_sessions, memory_used
  • GET /api/users — returns array with name, email, role, active
  • POST /api/users — create user (validates name, email, role)
  • POST /api/login — login (returns _form error on failure)
  • DELETE /api/users/3 — delete user #3
